# Database Backup System — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add an automated MariaDB backup container that dumps the database every 2h (8h–22h), stores backups locally (max 5 GB / 7 days) and uploads them encrypted to Google Drive (max 3 GB / 7 days).

**Architecture:** A dedicated `backup` Docker service (alpine + rclone + mariadb-client) added to `compose-prod.yaml`. A cron job inside the container runs `backup.sh` on schedule. rclone crypt handles AES-256 client-side encryption before upload to Google Drive.

**Tech Stack:** alpine:3.19, rclone, mariadb-client, busybox crond, jq, bash

### Task 2: backup.sh

**Files:**
- Create: `backup/backup.sh`

- [ ] **Step 1: Create `backup/backup.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail

# ── Config ────────────────────────────────────────────────────────────────────
BACKUP_DIR="/backups"
TIMESTAMP=$(date '+%Y-%m-%d_%H-%M')
FILENAME="mawster_${TIMESTAMP}.sql.gz"
MAX_LOCAL_BYTES=$((5 * 1024 * 1024 * 1024))   # 5 GB
MAX_REMOTE_BYTES=$((3 * 1024 * 1024 * 1024))  # 3 GB
RETENTION_DAYS=7
RCLONE_REMOTE="gdrive-crypt:mawster"

# ── 1. Dump ───────────────────────────────────────────────────────────────────
echo "[backup] $(date '+%Y-%m-%d %H:%M:%S') — Starting dump: $FILENAME"
mysqldump \
  --host=mariadb \
  --user=root \
  --password="${MARIADB_ROOT_PASSWORD}" \
  --databases "${MARIADB_DATABASE}" \
  --single-transaction \
  --routines \
  --triggers \
  | gzip > "${BACKUP_DIR}/${FILENAME}"

SIZE=$(du -sh "${BACKUP_DIR}/${FILENAME}" | cut -f1)
echo "[backup] Dump complete — size: $SIZE"

# ── 2. Local purge: by age ────────────────────────────────────────────────────
find "${BACKUP_DIR}" -name "mawster_*.sql.gz" -mtime "+${RETENTION_DAYS}" -delete
echo "[backup] Local: purged files older than ${RETENTION_DAYS} days"

# ── 3. Local purge: by size ───────────────────────────────────────────────────
while true; do
  TOTAL=$(du -sb "${BACKUP_DIR}" 2>/dev/null | cut -f1)
  [ "${TOTAL:-0}" -le "$MAX_LOCAL_BYTES" ] && break
  OLDEST=$(ls -t "${BACKUP_DIR}"/mawster_*.sql.gz 2>/dev/null | tail -1)
  [ -z "$OLDEST" ] && break
  echo "[backup] Local over limit ($(( TOTAL / 1024 / 1024 )) MB), deleting: $(basename "$OLDEST")"
  rm "$OLDEST"
done

# ── 4. Upload to remote ───────────────────────────────────────────────────────
echo "[backup] Uploading to remote..."
rclone copy "${BACKUP_DIR}/" "${RCLONE_REMOTE}/" --include "mawster_*.sql.gz"
echo "[backup] Upload complete"

# ── 5. Remote purge: by age ───────────────────────────────────────────────────
rclone delete "${RCLONE_REMOTE}/" --min-age "${RETENTION_DAYS}d" --include "mawster_*.sql.gz"
echo "[backup] Remote: purged files older than ${RETENTION_DAYS} days"

# ── 6. Remote purge: by size ──────────────────────────────────────────────────
REMOTE_SIZE=$(rclone size "${RCLONE_REMOTE}/" --json 2>/dev/null | jq -r '.bytes // 0')
while [ "${REMOTE_SIZE:-0}" -gt "$MAX_REMOTE_BYTES" ]; do
  OLDEST_REMOTE=$(rclone lsf "${RCLONE_REMOTE}/" --format "tp" --separator "|" --files-only 2>/dev/null \
    | sort | head -1 | cut -d'|' -f2)
  [ -z "$OLDEST_REMOTE" ] && break
  echo "[backup] Remote over limit ($(( REMOTE_SIZE / 1024 / 1024 )) MB), deleting: $OLDEST_REMOTE"
  rclone deletefile "${RCLONE_REMOTE}/${OLDEST_REMOTE}"
  REMOTE_SIZE=$(rclone size "${RCLONE_REMOTE}/" --json 2>/dev/null | jq -r '.bytes // 0')
done

echo "[backup] $(date '+%Y-%m-%d %H:%M:%S') — Done: $FILENAME"
```

### Task 3: restore.sh

**Files:**
- Create: `backup/restore.sh`

- [ ] **Step 1: Create `backup/restore.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail

BACKUP_DIR="/backups"
RCLONE_REMOTE="gdrive-crypt:mawster"

REMOTE=false
FILENAME=""

# ── Parse args ────────────────────────────────────────────────────────────────
while [[ $# -gt 0 ]]; do
  case "$1" in
    --remote)
      REMOTE=true
      shift
      ;;
    *)
      FILENAME="$1"
      shift
      ;;
  esac
done

if [ -z "$FILENAME" ]; then
  echo "Usage: restore.sh [--remote] <filename>"
  echo "Example: restore.sh mawster_2026-03-30_08-00.sql.gz"
  echo "Example: restore.sh --remote mawster_2026-03-30_08-00.sql.gz"
  exit 1
fi

# ── Download from remote if needed ───────────────────────────────────────────
if [ "$REMOTE" = true ]; then
  echo "[restore] Downloading $FILENAME from remote..."
  rclone copy "${RCLONE_REMOTE}/${FILENAME}" "${BACKUP_DIR}/"
  echo "[restore] Download complete"
fi

FILEPATH="${BACKUP_DIR}/${FILENAME}"

if [ ! -f "$FILEPATH" ]; then
  echo "[restore] ERROR: File not found: $FILEPATH"
  echo "[restore] Available local backups:"
  ls -lh "${BACKUP_DIR}"/mawster_*.sql.gz 2>/dev/null || echo "  (none)"
  exit 1
fi

# ── Restore ───────────────────────────────────────────────────────────────────
echo "[restore] Restoring from $FILENAME..."
gunzip -c "$FILEPATH" | mysql \
  --host=mariadb \
  --user=root \
  --password="${MARIADB_ROOT_PASSWORD}"

echo "[restore] Done."
```

### Task 4: rclone.conf.example

**Files:**
- Create: `backup/rclone.conf.example`

- [ ] **Step 1: Create `backup/rclone.conf.example`**

```ini
# ─────────────────────────────────────────────────────────────────────────────
# rclone.conf — Google Drive + crypt remote configuration
#
# HOW TO GENERATE THE REAL FILE ON THE SERVER:
#   1. Install rclone on the host:  curl https://rclone.org/install.sh | sudo bash
#   2. Run: rclone config
#   3. Create a remote named "gdrive" of type "drive" (follow OAuth flow)
#   4. Create a remote named "gdrive-crypt" of type "crypt" on top of "gdrive"
#      - Remote: gdrive:mawster-backups
#      - filename_encryption: standard
#      - directory_name_encryption: true
#      - Set a strong password + salt (save them somewhere safe — loss = data loss)
#   5. Copy the resulting ~/.config/rclone/rclone.conf to ./backup/rclone.conf
#   6. Never commit rclone.conf (it is gitignored)
# ─────────────────────────────────────────────────────────────────────────────

[gdrive]
type = drive
client_id = YOUR_OAUTH_CLIENT_ID
client_secret = YOUR_OAUTH_CLIENT_SECRET
scope = drive
token = {"access_token":"...","token_type":"Bearer","refresh_token":"...","expiry":"..."}

[gdrive-crypt]
type = crypt
remote = gdrive:mawster-backups
filename_encryption = standard
directory_name_encryption = true
password = YOUR_RCLONE_CRYPT_PASSWORD
password2 = YOUR_RCLONE_CRYPT_SALT
```

## Server Setup (post-deploy, one-time)

These steps are done manually on the VPS after deploying, not automated:

1. Install rclone on the **host**: `curl https://rclone.org/install.sh | sudo bash`
2. Run `rclone config` on the host to generate the real `rclone.conf` (OAuth + crypt setup)
3. Copy the result to `./backup/rclone.conf` in the project directory
4. Run `docker compose -f compose-prod.yaml up -d --build backup`
5. Verify first backup manually: `docker exec backup /usr/local/bin/backup.sh`
6. Check logs: `docker logs backup`
